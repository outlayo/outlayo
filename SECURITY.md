# Security Policy

## Supported Surface

This repository contains the public OSS runtime, connectors, SDKs, CLI flows, and documentation for Outlayo.

Do not report issues related to:

- hosted production infrastructure
- private operational controls
- customer data exposure in non-public systems

Those systems are intentionally outside this repository.

## Reporting a Vulnerability

Please do not open a public GitHub issue for suspected security vulnerabilities.

Instead, report them privately to: `security@outlayo.com`

Include:

- affected component or file path
- reproduction steps or proof of concept
- impact assessment
- any suggested mitigation if available

We will acknowledge receipt as quickly as possible, aim to triage within 5 business days, and coordinate a fix before public disclosure.

## Disclosure Expectations

- use private reporting first
- avoid accessing or modifying data that does not belong to you
- do not publish exploit details until a fix is available or coordinated disclosure is agreed

## Secrets Hygiene

If you believe credentials were exposed in this repository or its history, report that privately as well so rotation guidance can be coordinated.
